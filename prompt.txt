지금부터 일정관리 웹 기반 프로그램을 하나 만든다. \
백엔드는 Spring Boot, 프론트는 next.js 혹은 react를 활용한다. \
\
<주의 사항> \
사용자가 해당 프로그램에 추가하는 일정 및 모든 자료들은, 보안을 위해 절대 서버가 아닌 사용자의 로컬에 저장되어야 한다. \
\
<주요 개념>\
0. Cycle: 전체 프로젝트에 대한 단위라고 이해하면 된다. \ 모든 Goal / Work / Task들은 하나의 Cycle 내에서 생성되고 관리된다. \ 
1. Goal: Cycle 내에서 가장 큰 단계의 개념이다. \
최종적으로 이루고자 하는 목표를 나타낸다. \
각 Goal은 하위 개념으로 Work을 가질 수 있다. 모든 Goal은 상태를 가지며, 시작 전 / 진행 중 / 완료 라는 세 개의 값 중 하나를 가진다. \
Goal의 상태는 하위 Work들의 상태에 의존한다. 하위 Work가 모두 '시작 전' 상태이면 Goal의 상태는 '시작 전'이다. 하위 Work가 모두 '완료' 상태이면 Goal의 상태는 '완료'이다. 이외에는 Goal의 상태는 모두 '진행 중'이다. \
Goal을 생성할 때 사용자가 목표 시작일 / 목표 완료일 을 설정 가능하다. \
\
2. Work: Goal 다음으로 큰 개념이다. \
Goal을 이루기 위해 진행시켜야 하는 일들을 나타낸다. \
다만, 모든 Work가 꼭 Goal을 가져야 하는 것은 아니다. Goal이 없이도 Work를 생성할 수 있다. Work를 생성할 때, 필요에 따라 이미 생성되어 있는 Goal을 고를 수 있다. \
Work의 상태는 사용자가 임의로 변경 가능하며, 시작 전, 진행 중, 완료 세 개 중 고를 수 있다.\
Work을 생성할 때 사용자가 목표 시작일 / 목표 완료일 을 설정 가능하다. \
\
3. Task: 가장 작은 개념이다. \
Work를 완료시키기 위해 실제로 진행해야 하는 행위를 나타낸다. \
Task를 생성할 때에는 반드시 상위 Work가 선택되어야 한다. \
Task의 상태는 완료/미완료 두 가지 밖에 없으며, 간단히 체크 박스로만 표현한다. \
필요 시 목표 완료 일을 지정할 수 있다. \
\
<프론트 엔드> \
Cycle의 생성 및 선택은 좌측 상단에서 드롭박스를 내려서 만들거나 생성할 수 있다. \
하나의 Cycle 내에서는 자체적으로 세 개의 탭을 가지며, 각 탭은 보여지는 방식만 다르고 동일한 데이터들을 보여준다. \
탭은 화면 하단에서 고를 수 있다. \
1. Goal List \
각 Goal들에 대해 Goal - Work - Task 들을 볼 수 있는 탭이다. Task는 처음에는 접혀있는 상태이며, 필요하면 Work별로 펼쳐서 볼 수 있다. \
각 Goal들은 카드 형태로 표기된다. \
Goal마다 진행률, 상태, 목표 시작일, 목표 완료일이 표기된다. \
진행률의 경우, 완료처리된 하위 Work의 비율로 계산한다. \
탭 한쪽에 '완료된 항목들 가리기' 버튼과 체크박스가 있다. 해당 버튼을 체크할 경우, 완료된 Goal / Work / Task는 모두 가려진다. \
2. 캘린더 \
캘린더 화면이다. \ 캘린더 상에는 Goal / Work / Task들의 목표 시작일, 목표 완료일이 표기된다. \
미완료된 항목은 하늘색으로, 완료된 항목은 연한 회색으로 나타난다. \
3. To-Do List \
한 달 이내에 완료해야 하는 Task들만 Work 별로 정리해서 보여준다. 각 Task는 체크박스로 완료처리가 가능하다. \
Goal List 화면과 마찬가지로, 완료된 항목들 가리기 체크박스가 있다.
